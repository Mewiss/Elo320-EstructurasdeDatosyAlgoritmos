Valgrind

Herramienta para detectar errores de memoria en programas C/C++.

  • Detecta memory leaks, accesos inválidos, uso de memoria no inicializada, etc.
  • Proporciona informes detallados para ayudar a localizar y corregir errores.

Valgrind

Uso básico:


valgrind --leak-check=full ./mi_programa
  • --leak-check=full: Modo detallado para detectar memory leaks.
  • ./mi_programa: Ejecutable a analizar.

Requiere que el programa se compile con la opción -g para incluir información de depuración.

Valgrind

Ejemplo de salida:


==1234== Invalid write of size 1
==1234==    at 0x1091A3: main (arreglo.c:23)
==1234==  Address 0x5203040 is 0 bytes after a block of size 40 alloc'd
==1234==    at 0x4C2F1C3: malloc (vg_replace_malloc.c:307)

Indica un acceso inválido a memoria, con detalles sobre la ubicación del error (arreglo.c:23)

Ejemplo de salida:


==1256== 24 bytes in 2 blocks are definitely lost in loss record 1 of 1
==1256==    at 0x4C2F1C3: malloc (vg_replace_malloc.c:307)
==1256==    by 0x10915B: insertar (lista.c:18)
==1256==    by 0x1091F2: main (lista.c:41)

Indica memory leaks, con detalles sobre dónde se asignó la memoria que no fue liberada (lista.c:18)
