InsertSort

Es un método de ordenamiento de elementos, en el cual a través de la visitas de subconjuntos, un elemento es insertado en la posición ordenada que le corresponde.

Ejemplo: Insertion Sort ($\mathcal{O}(n^2)$)

Inicio:
12 11 13 5
Iteración $i=1$:
11 12 13 5
Iteración $i=2$:
11 12 13 5
Iteración $i=3$:
5 11 12 13
  • Iteración $i=1$: $\text{Key}=11$. El 11 es menor que 12. Se desplaza 12 a la derecha y se inserta 11 al inicio.
  • Iteración $i=2$: $\text{Key}=13$. El 13 es mayor que el último elemento ordenado (12). No hay desplazamientos.
  • Iteración $i=3$: $\text{Key}=5$. El 5 es menor que todos los elementos ordenados. Se desplazan los tres elementos a la derecha y se inserta 5 al inicio.

InsertSort

¿Cuál es la complejidad del siguiente algoritmo?


void insertionSort(int arr[], int n){
    int i, key, j;
    for (i = 1; i < n; i++) {
        key = arr[i];
        j = i - 1;
        
        // Mover elementos de arr[0..i-1], que son mayores que key,
        // a una posición adelante de su posición actual
        while (j >= 0 && arr[j] > key) { //esto puede ser un for
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}

InsertSort: Complejidad

  • Cota Superior: $\mathcal{O}(n^2)$
    (Caso peor: todo invertido).
  • Cota Inferior: $\Omega(n)$
    (Mejor caso: todo ordenado).
  • Cota Promedio: $\Theta(n^2)$
    (Caso promedio para arreglos aleatorios).

Merge Sort

Algoritmo de ordenamiento recursivo que aplica la técnica Divide & Conquer (Divide y Vencerás).

Características

  • Es Estable y rápido para grandes tamaños de datos.
  • Generalmente es superado por Quick Sort en la práctica.
  • Necesita un espacio de memoria extra para el ordenamiento ($\mathbf{\Theta(n)}$).

Ejemplo: Merge Sort

Paso 0: Inicio

3 8 2 5 1 4 7 6

Paso 1: Dividir (Nivel 1)

3 8 2 5
1 4 7 6

Paso 2: Dividir (Nivel 2 - hasta llegar a elementos unitarios)

3 8
2 5
1 4
7 6

Ordenamos

3 8
2 5
1 4
6 7
  • Fase 1 (Divide): El arreglo se divide en dos mitades. El costo de división es $\mathcal{O}(1)$.

Ejemplo: Merge Sort

Paso 3: Mezclar (Nivel 2 - Costo $\mathcal{O}(n)$ total)

3 8 2 5
1 4 6 7

Ordenar

2 3 5 8
1 4 6 7

Paso 4: Mezcla Final

1 2 3 4 5 6 7 8
  • Fase 2 (Merge): Las sublistas de tamaño 1 se mezclan para formar listas ordenadas de tamaño 2, luego 4. Este trabajo lineal se realiza en cada nivel.

MergeSort

Función mergeSort

void mergeSort(int arr[], int l, int r){ //l=izq, r=der (indices)
    if (l < r) {
        int m = l + (r - l) / 2; //div. entera

        mergeSort(arr, l, m);    // T(n/2) - dividir
        mergeSort(arr, m + 1, r); // T(n/2) - post-orden

        merge(arr, l, m, r);    // O(n) - mezclar
    }
}

Relación de Recurrencia: $\mathbf{T(n) = 2T(n/2) + \mathcal{O}(n)}$

MergeSort

Función merge (La clave de $\mathcal{O}(n)$)

void merge(int arr[], int l, int m, int r){
    int i, j, k;
    int n1 = m - l + 1;
    int n2 = r - m;

    int L[n1], R[n2]; //arreglos temporales (Costo de espacio)

    for (i = 0; i < n1; i++) 
        L[i] = arr[l + i];
    for (j = 0; j < n2; j++)
        R[j] = arr[m + 1 + j];

    i = 0; j=0; k = l; //mezcla
    while (i < n1 && j < n2) {
        if (L[i] <= R[j])
            arr[k] = L[i]; i++;
        else
            arr[k] = R[j]; j++;
        k++;
    }

    while (i < n1) //si sobra de L
        arr[k] = L[i]; i++; k++; 
    while (j < n2)  //si sobra de R
        arr[k] = R[j]; j++; k++;
}

QuickSort

Es un algoritmo de ordenamiento no estable de complejidad promedio esperada $\Theta(n \log n)$, sin embargo, puede en sus peores casos llegar a $\mathcal{O}(n^2)$. Al igual que MergeSort utiliza la técnica de Divide & Conquer.


QuickSort es uno de los algoritmos más implementados en estándares de lenguajes actualmente:

  • Usa recursividad.
  • No usa estructuras de datos anexas (es in-place).
  • Técnicas para evitar complejidad $\mathcal{O}(n^2)$ (pivote aleatorio, mediana de 3, etc).
  • Naturalmente modelado para disminuir complejidad.

QuickSort: Tres Pasos

  1. Dividir (Partition): Separa el arreglo en dos sub-arreglos (particiones) alrededor de un pivote, donde los miembros de uno son siempre menor que el pivote y los del otro son mayores.
  2. Conquistar: Ordenar recursivamente los sub-arreglos generados.
  3. Combinar: Como la partición inicial garantiza que los sub-arreglos ya están en el lado correcto del pivote, la combinación es una tarea trivial (no se requiere trabajo adicional).

  • Fase 3 (Combinar): En Quick Sort, la combinación es trivial. Como los pivotes ya están en su lugar final, no se requiere trabajo adicional.

Esquemas de Partición

QuickSort se basa en un esquema de particiones que puede ser implementado de variadas formas. Estos esquemas dependen de un valor pivote, que permite ordenar los sub-arreglos.


Generalmente, se comparan dos de las formas más utilizadas:

  • Forma Lomuto: Pivote al final. Fácil implementación. A menudo más lento debido a intercambios redundantes.
  • Forma Hoare: Forma original. Pivote al inicio. Sofisticado. Muy eficiente (requiere menos intercambios en promedio).

Partition: Hoare's Scheme (Implementación)

CÓDIGO PARTITION 
int partition(int arr[], int inicio, int fin){
    int pivote = arr[inicio];
    int i = inicio - 1, j = fin + 1, temp;
    
    while (i < j){
        do{
            i++;
        }while (arr[i] < pivote);
        
        do{
            j--;
        }while (arr[j] > pivote);

        if(j <= i)
            break;
        
        // Intercambio
        temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }

    return j; // Retorna el punto de corte
}

QuickSort (Función Principal)

Quicksort en sí es un procedimiento sencillo y recursivo.

CÓDIGO QUICKSORT 
void quickSort(int arr[], int inicio, int fin){
    if (inicio < fin) {
        
        // 1. Divide: Generar el punto de corte (j)
        int corte = partition(arr, inicio, fin);
        
        // 2. Conquistar: Llamada recursiva a la partición izquierda
        quickSort(arr, inicio, corte);
        
        // 3. Conquistar: Llamada recursiva a la partición derecha
        quickSort(arr, corte + 1, fin);
    }
}

QuickSort Aleatorizado

¿Cuál es el peor escenario de QuickSort?

Es cuando siempre nos queda una partición de 1 y otra de $n-1$ (ej. el pivote es siempre el mínimo o el máximo).

  • Prácticamente tendremos que recorrer completamente el arreglo $n$ veces.
  • La recursión no será de largo $\log n$ sino $n$.
  • Por lo tanto, la complejidad en el peor escenario es $\mathbf{\mathcal{O}(n^2)}$.

Para evitar esto, la elección del pivote se hace de forma aleatoria (o usando la mediana de tres), así el tiempo esperado (concepto estadístico) del peor escenario es $\mathcal{O}(n \log n)$. (Nota: el peor caso determinista sigue siendo $\mathcal{O}(n^2)$).

QuickSort Aleatorizado (Función Principal)

QUICKSORT ALEATORIZADO 
void quickSort(int arr[], int inicio, int fin){
    if (inicio < fin) {
        
        // Se llama a la función de partición que elige el pivote aleatoriamente
        int corte = random_partition(arr, inicio, fin);
        
        quickSort(arr, inicio, corte);
        quickSort(arr, corte + 1, fin);
    }
}

Random Partition (Partición Aleatoria)

RANDOM PARTITION 
int random_partition(int arr[], int inicio, int fin){
    // 1. Elegir un índice aleatorio entre inicio y fin
    int index = rand() % (fin - inicio + 1) + inicio; 
    
    // 2. Intercambiar el pivote elegido (index) con el inicio
    int temp_swap = arr[index];
    arr[index] = arr[inicio];
    arr[inicio] = temp_swap;

    // 3. Llamar al esquema de partición de Hoare con el nuevo pivote en 'inicio'
    return partition(arr, inicio, fin); 
}

// Nota: La función partition ahora usa el elemento en 'inicio' como pivote,
// que ya fue aleatorizado por 'random_partition'.
