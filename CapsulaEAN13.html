Codificación EAN-13

El código se divide lógicamente para identificar el origen y el producto mediante 13 dígitos.

Dígitos Nombre Función
1 al 3 Código de País (Prefijo GS1) Identifica el país o región de la organización.
4 al 12 Código de Empresa y Producto Identifica la empresa y el número único del artículo.
13 (Final) Dígito de Verificación (Control) Se calcula usando los primeros 12 dígitos para asegurar que no haya errores de lectura.

Codificación EAN-13

El EAN-13 usa anchos de barra (módulos) para codificar 13 dígitos.

  • Caracteres especiales: Inicio (101), Centro (01010), Fin (101).
  • Caracteres de datos: 6 a la izquierda y 6 a la derecha.
  • El primer dígito (D13) no se codifica directamente.

Codificación de los datos

Cada dígito se representa como una secuencia de barras y espacios de 7 módulos.

Ejemplo de secuencia

2 - 1 - 2 - 2

1101100

barra(x2) - espacio (x1) - barra(x2) - espacio(x2)

Total: 7 módulos. Este patrón se busca en la tabla.

Codificación de los datos

Un mismo dígito puede tener hasta tres patrones de barras diferentes (L, G o R).

El patrón '2122' o '1101100' siempre codifica un dígito único (en este caso, '2').

Patrón '2122' con Paridad R

1101100

Ejemplo: Dígito '2' en la mitad derecha.

Patrón '2122' con Paridad L

0010011

Ejemplo: Dígito '2' en la mitad izquierda.

Codificación por Paridad

Paridad L (Left)
  • Patrones "claros".
  • Ejemplo (Dígito 4): 0100011

Paridad G (Group)

  • Patrones "oscuros" (con más barras).
  • Ejemplo (Dígito 4): 1011100

Paridad R (Right)

  • Es el inverso binario de L.
  • Ejemplo (Dígito 4): 1011100 (Inverso de L)

Cálculo del Dígito de Verificación (D13)

Asegura la precisión de la lectura.

Paso 1: Suma Ponderada

  1. Suma los dígitos en posiciones pares (D2, D4, D6, etc.) y multiplica el resultado por $\times 3$.
  2. Suma los dígitos en posiciones impares (D1, D3, D5, etc.).
  3. Suma los dos resultados.

Ejemplo: $S = (\text{Pares} \times 3) + \text{Impares}$

Cálculo del Dígito de Verificación (D13)

Paso 2: Cálculo Final

Se busca el dígito que, al sumarse a $S$, hace que el resultado sea un múltiplo de 10.


// Si la suma total es 73:
int modulo = S % 10; // 73 % 10 = 3
int verificador = (10 - modulo) % 10; 
// (10 - 3) % 10 = 7. El dígito es 7.

Si el escáner lee los 13 dígitos y recalcula el control, debe coincidir. Si no coincide, ¡error de lectura!

Extracción del Patrón: El Módulo

El primer paso es pasar de una imagen a una secuencia de números. La unidad de medida es el Módulo (el ancho de barra más fino).

Medición

Se mide el ancho de cada barra y espacio.

Resultado: Una secuencia de anchos.

Normalización

Divide todos los anchos por la unidad mínima (el módulo). El resultado siempre es 1, 2, 3 o 4.

Resultado: La secuencia de anchos de módulo.

Eficiencia: $O(\log N)$ vs. $O(1)$

Opción Lenta: Árbol Binario (ABB)

  • Necesita una clave compleja (Patrón + Paridad).
  • Tiempo: $O(\log N)$.

Opción Ideal: Tabla Hash

  • Mapea la clave "Dígito + Paridad".
  • Tiempo: $O(1)$ (Constante).

Implementación: La Tabla Hash en C

La Tabla Hash se modela con arreglos, usando el dígito como índice para acceso inmediato.


// Patrones de Paridad 'L' (Izquierda)
// Índice [0] -> "0001101", Índice [1] -> "0011001", etc.
const char *Paridad_L[] = { /* ... 10 patrones L ... */ };

// Patrones de Paridad 'R' (Derecha)
const char *Paridad_R[] = { /* ... 10 patrones R ... */ };

char* buscar_patron_O1(int digito, char region) {
    // La búsqueda es un acceso directo: O(1)
    if (region == 'I') { 
        return (char*)Paridad_L[digito]; 
    } else {
        return (char*)Paridad_R[digito];
    }
}

Resumen

  • El EAN-13 codifica 13 dígitos usando patrones de barras y espacios.
  • El primer dígito se representa mediante la paridad de los siguientes 6 dígitos.
  • El dígito de verificación asegura la precisión de la lectura.
  • La decodificación eficiente usa tablas hash para acceso en tiempo constante $O(1)$.
