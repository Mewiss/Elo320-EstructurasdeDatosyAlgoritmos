HeapSort

Algoritmo de ordenamiento basado en la estructura de datos Heap (Montículo). Utiliza un Max Heap para ordenar los elementos extrayendo repetidamente el máximo y reconstruyendo el heap.

  • Complejidad: $\mathbf{\mathcal{O}(n \log n)}$ en todos los casos (mejor, promedio y peor).
  • In-place: Requiere espacio adicional constante $\mathbf{\mathcal{O}(1)}$.
  • No Estable: No mantiene el orden relativo de elementos iguales.

HeapSort: Heapify

Función Clave para Mantener la Propiedad Heap


void heapify(int arr[], int i, int tam){ // arreglo, índice, tamaño
    int mayor = i; // al inicio el nodo es el mayor
    int l = hijo_izq(i); 
    int r = hijo_der(i); 

    // 1. Comparar con el hijo izquierdo
    if (l < tam && arr[l] > arr[mayor]) 
        mayor = l; 
    
    // 2. Comparar con el hijo derecho
    if (r < tam && arr[r] > arr[mayor]) 
        mayor = r;

    // 3. Intercambio y recursión
    if (mayor != i){ // si el mayor no es el padre original
        // Realizar el intercambio
        int temp = arr[i];
        arr[i] = arr[mayor];
        arr[mayor] = temp;

        // Llamada recursiva en el sub-árbol afectado
        heapify(arr, mayor, tam); 
    }
}

HeapSort: Build Heap 


void build_heap(int arr[], int tam){ // arreglo y tamaño

    // Empezamos desde el último nodo interno (tam/2) hasta la raíz (1 o 0)
    for (int i = tam / 2; i >= 0; i--){ 
        heapify(arr, i, tam);
    }
}

Eficiencia: La construcción del heap es sorprendentemente eficiente, con una complejidad de $\mathbf{\mathcal{O}(n)}$.

HeapSort (Código)


void HeapSort(int arr[], int tam){ // arreglo y tamaño

    int temp;

    // Fase 1: Construir el Max Heap
    build_heap(arr, tam);

    // Fase 2: Extraer elementos uno por uno
    for (int i = tam - 1; i > 0; i--){ // 'i' actúa como el nuevo tamaño del heap
        
        // 1. Intercambiar la raíz (max) con el último elemento del heap
        temp = arr[0]; 
        arr[0] = arr[i];
        arr[i] = temp;

        // 2. Arreglar el heap (el elemento más grande fue movido)
        heapify(arr, 0, i); 
    }
}

Complejidad: HeapSort es $\mathbf{\mathcal{O}(n \log n)}$ en el mejor, peor y caso promedio, ya que $\log n$ es el costo de cada llamada a `heapify`.

Colas de Prioridad (Priority Queues)

La estructura de Heap Binario (Binary Heap) es la base de las Colas de Prioridad, donde los datos se asocian con un valor entero que representa su prioridad.

Aplicaciones Comunes:

  • Schedulers de clúster: Manejo de trabajos con diferentes niveles de importancia (e.g., `qsub`, `sbatch`).
  • Prioridad de procesos: Gestión de la asignación de CPU en sistemas operativos (Task Manager).
  • Algoritmos de grafos como el de Dijkstra (para encontrar el camino más corto).

Colas de Prioridad: Operaciones y Costos

Operaciones Básicas

  • `get_heap`: Retorna el valor del máximo valor en la cola.
  • `pop_heap`: Retorna y remueve el valor máximo en la cola.
  • `increase_key`: Actualiza la prioridad de un elemento, y reordena el heap.
  • `insert_key`: Añade un nuevo valor a la cola, y la reordena.

Complejidad Algorítmica

  • `get_heap`: $\mathbf{\mathcal{O}(1)}$ (acceso directo a la raíz).
  • `pop_heap`: $\mathbf{\mathcal{O}(\log n)}$ (costo de `heapify`).
  • `increase_key`: $\mathbf{\mathcal{O}(\log n)}$ (subir el elemento por el heap).
  • `insert_key`: $\mathbf{\mathcal{O}(\log n)}$ (bajar o subir el elemento por el heap).

Algoritmo Avanzado: Backtracking (Vuelta Atrás)

Técnica algorítmica utilizada para encontrar soluciones a problemas de decisión u optimización que involucran la exploración de un gran espacio de estados (ej. rompecabezas, laberintos, combinaciones).

¿Cómo funciona?

  1. Hacer una Suposición: Se elige una opción para el estado actual.
  2. Llamada Recursiva: Se avanza a la siguiente etapa con el cambio.
  3. Verificación: Si la suposición falla (viola restricciones o no conduce a la meta).
  4. Vuelta Atrás: Se deshace el cambio y se prueba la siguiente opción disponible (el corazón del Backtracking).

Es una búsqueda en profundidad (DFS) que poda ramas inválidas del árbol de búsqueda.

Aplicaciones Clave

  • Sudoku Solver: El ejemplo más clásico.
  • Problema de las N-Reinas: Colocar N reinas sin que se ataquen.
  • Laberintos: Encontrar una salida.
  • Optimización: Problema de la Mochila (Knapsack).
  • Generación de Combinaciones o Permutaciones.
